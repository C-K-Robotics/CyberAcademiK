<Section id="loop" n="01" title="Open loop vs. closed loop" chip="Control loop">

In **open-loop** control the command ignores the result — simple and cheap, but
blind. A **closed loop** feeds the measured output back, comparing it to the
setpoint and acting on the *error*. PID lives inside the controller block below.

<Section id="plant" n="02" title="The plant — a mechanical system" chip="The plant">

Most robot mechanisms reduce to three elements: a **mass**, a **spring**, and a
**damper**. Apply a force and the position obeys a second-order differential
equation. Pluck it to feel its natural, damped response.

<Section id="tune" n="05" title="Tuning — Ziegler–Nichols" chip="Tuning">

Raise Kp (with Ki = Kd = 0) until the loop oscillates steadily — that's the
**ultimate gain Ku** at period **Tu**. Plug them in and the classic rules give you
a starting tune.

<Section id="code" n="06" title="Implementing PID in code" chip="Code">

The discrete form runs every loop iteration (here ~20 ms): compute the error,
accumulate the integral, take the derivative, and sum the weighted terms into a
motor command.

<CodeTabs>
<CodeTab label="Drive.java">

```java
public class Drive {
    double P = 0.6, I = 0.02, D = 4.0;
    double integral, previousError, setpoint = 0;
    Gyro gyro;
    DifferentialDrive robotDrive;

    public Drive(Gyro gyro) {
        this.gyro = gyro;
    }

    public void setSetpoint(double setpoint) {
        this.setpoint = setpoint;
    }

    public void pid() {
        double error = setpoint - gyro.getAngle();   // error = target - actual
        this.integral += error * 0.02;               // integral grows by error * dt (20 ms)
        double derivative = (error - previousError) / 0.02;
        double rcw = P * error + I * integral + D * derivative;
        previousError = error;
        robotDrive.arcadeDrive(0, rcw);
    }
}
```

</CodeTab>
<CodeTab label="Drive.cpp">

```cpp
class Drive {
    double P = 0.6, I = 0.02, D = 4.0;
    double integral = 0, previousError = 0, setpoint = 0;
    Gyro& gyro;
    DifferentialDrive& robotDrive;

public:
    Drive(Gyro& g, DifferentialDrive& d) : gyro(g), robotDrive(d) {}

    void setSetpoint(double sp) {
        this->setpoint = sp;
    }

    void pid() {
        double error = setpoint - gyro.getAngle();   // error = target - actual
        this->integral += error * 0.02;              // integral grows by error * dt
        double derivative = (error - previousError) / 0.02;
        double rcw = P * error + I * integral + D * derivative;
        previousError = error;
        robotDrive.arcadeDrive(0, rcw);
    }
};
```

<Section id="arm" n="07" title="Arm simulator — beating gravity" chip="Arm simulator">

A real mechanism rarely behaves like a frictionless plant. Here a **Falcon 500**
drives a pivoting arm that **gravity constantly pulls down** — the torque needed
to hold it changes with angle. P alone leaves a droop; switch on a gravity
**feed-forward** term and PID is freed to correct only what's left.
